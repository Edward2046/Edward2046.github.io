规规矩矩做狗

By EdwardGuo

發表於 2016-02-13

回来的路上看到一只微笑天使–萨摩耶不听主人话到处乱嗅乱跑被主人训斥，便想起当初自己对Blue也常常发脾气，也因为他不听话，到处乱嗅，见到狗就激动不已，特别是母狗。中国人越来越接收宠物特别是狗和猫，这是个好现象，说明大部分国人都是富有爱心，同时客观的讲，大部分中国的宠物由于缺乏合适的培训（矫情点说是教育），导致了主人会觉得力不从心，以下总结下中国宠物狗的不良习惯（作为一个狗爸，绝对没有半点讽刺宠物狗）
1.导出乱嗅：这可能最常见并且最令狗狗烦恼的一个不良习惯，虽说是狗的本性，发现同类的味道会去闻一下，但是在遛狗的时候倘若狗狗能抬头往前走那多么愉快。
2.随地排便：虽然大部分主人会自带塑料袋收拾狗狗的排便，但是毕竟不能完全清楚狗狗的排便，特别狗狗随处乱撒尿，给路人会带来不快，也给环境带来不良影响。
3.见到狗便激动：绝大部分狗见到同类不是激动的乱跳就是霸道的乱叫，遛狗时让大部分主人感到心累，倘若狗见到同类能安静的上次打个招呼那多美。
4.扑人：当遇到热情的爱狗人士时，会激动扑上去，些许时候会吓到别人，倘若遇到热情的爱狗人士能淡定舔舔对方的手那该多好。
5.挑食：大部分狗吃了好的东西便不爱吃狗粮，主人饿着又心疼，给吃肉又贵对狗本身又不好，这也是一大痛点。
6.打架：打架也是很多大狗的恶习，金毛，拉布拉多，萨摩尤为常见，大狗常常为了争抢底盘或博取母狗的欢心而打架，虽说是动物的本能，但毕竟是宠物狗，不能按照本能去要求。

以上是国内宠物狗的不良习惯，当然还有很多很规矩的狗狗，如果狗狗都能规规矩矩，那会给城市生活带来更多乐趣，也给主人减少很多烦恼，一条理想的规矩的狗应该是：
1.性格温顺，不霸道，与狗为善。
2.不容易激动，不管见到人还是狗。
3.抬头走路，抵抗住同类味道的诱惑不到处乱嗅。
4.不随地大小便，能克制，养成规律的排便习惯。
5.不挑食，不乱吃东西。
6.喜欢和孩子玩耍，培养孩子的爱心。

如何让狗变得规矩，养成好的习惯是一个大难题，毕竟狗爸狗妈们都没有太多专业的训练知识，就算网上找到了相关的视频，有时候也因太忙没时间，没耐心坚持去训练。倘若有门槛较低的宠物学校，让家养宠物狗狗得到相关专业的培训，于人于狗都是件好事，但目前还只是个期许，当然决定狗狗规不规矩的最根本因素还是狗主人，愿更多的宠物狗都能得到适当地培训，规规矩矩做狗，成为狗爸狗妈的掌上明珠。

2016该学点什么(上)？

By EdwardGuo

發表於 2016-02-13 

春节假期结束，开始2016的海漂征程，每年年初都会立志新一年里要多学习，但是学什么脑海里从来没有一个清晰的概念，总是得临时去想，而没有一个清晰的学习路线，今天花点时间梳理下2016年自己要学习的内容。
一，技术-生存之本
    作为一个小白，总结一个互联网公司需要的技术：
    1.后端语言：Java/php，作为一个服务端工程师，Java语言是一切系统的运行之本。
    2.代码管理：git,团队开发的必备免费工具。
    3.程序部署：tomcat,如何发布程序，包括web程序和服务端程序。
    4.服务调用：pegion/dubbo/thrift，Java下的RPC框架，通俗讲就是统一管理配置各个系统服务间的调用，起到服务治理的作用，具体怎么实现，我不知道。
    5.Linux:   需要掌握基本的linux操作命令和常用的的shell脚本命令。
    6.IOS/Android: 在移动的趋势下，互联网公司必须需要IOS/Anroid开发工程师，Swift会是IOS开发的一个趋势。
    7.前端开发：HTML5已经被越来越多的采纳了，所以掌握HTML5对一家互联网公司也极为重要。
    8.数据库：  MySQL已经成为绝大部分互联网公司的必备免费开源数据库，掌握基本的SQL语句，索引优化无可厚非。
    9.缓存知识：memcache或redis,缓存在某些情况能极大降低数据库的压力，学会如何使用缓存在关键时刻能带来不错的效果。
    10.网络通信：网络通信对于互联网公司之间的通信也是必备的，掌握Http以及Socket的知识可以不必公司之间进行对接时抱佛脚。
以上便是加入互联网1年半以来的切身总结，可能不够全，但基本覆盖，那自己该在哪些知识上加强？个人对业务还比较感兴趣，所以还是需要在离业务更近的知识领域加强，即Java/Http/IOS/H5/MySQL.

    Java的提升计划：
    1）习惯性总结：对于每次的新发现及时总结，从事Java开发许久，但是知识都是零碎的，一直没有形成一个体系，2016学会定期写日志总结。
    2）Java注释和反射：Java的Annotation和Reflect这2个包一直以来是个盲区，2016年上半年集中优先学习下，并且形成日志文档。
    3）代码工具化：学会将常用的代码工具化，形成通用的模块，减少重复的代码开发。
    4）Java虚拟机：JVM一直以来是弱点，了解JVM可以更好地应对系统出现的异常，及时定位并且解决问题，JVM的内存分布基本清楚，需要进一步了解Java对象的生命周期。
    5）系统设计： 总结些合理的系统设计方案，如何更加合理的解耦，同时又不至于过分的拆分。

    Swfit提升计划：
    1）Swift空间布局：学会CollectionView控件和TableView控件的布局。
    2）Swift调用服务：学会如何Swfit和Java的通信。

    MySQL提升计划：
    1）分库分表：2016年MySQL的主要提升计划是分库分表，如何将一张大表分成多张小表，并且做到顺利的灰度切换，2016年把这个做好即可。

Linux下循环使用Curl命令

By EdwardGuo

發表於 2016-01-31 

1.关于curl
再说curl有必要了解下libcurl,摘自wikipedia "libcurl is a free client-side URL transfer library, supporting, FTP, FTPS, HTTP, HTTPS, SCP, Telent..",也就是这说libcurl是个URL的传输工具包，支持各种协议的传输并且是线程安全，而curl不过是libcurl的一个命令而已，curl最常用的方式便是curl www.example.com 用于获取该url的网页内容。
以下为curl的常用参数
curl -o /tmp/sina.html http://www.sina.com.cn/ : 将新浪主页的内容抓取下来并存放到本地的tmp目录下。
curl -I http://www.sina.com.cn/ :只显示文档信息，不显示全部内容，主要用于判断该url是否OK。
curl -X POST http://www.sina.com.cn/: curl默认的http请求方式是GET，-x参数可以改变http请求方式。

关于curl的常用使用方式就这些，下面看看如何在linux下循环使用curl,主要是linux中while和for循环的嵌套使用。

2.while循环和for循环的嵌套使用。
废话不多说，来个实例场景，我想看看大众点评网是不是每个城市都有对应的主页，思路如下：
    a.通过while循环读取城市列表，每一行中有多个城市通过逗号分隔。
    b.在while循环中，每读一行，for循环读取这一行中的每一个城市。
    c.每读一个城市通过curl -I来获取返回HTTP请求信息。
    d.打印结束，注意echo 后要加一个空格。

看看代码：

#!bin/bash
#curl的小工具
#cityList="shanghai,dalian,beijing,qingdao,nanchang,taiyuan"
RESULTFILE="/tmp/HttpCityCodeNew.txt"
IFS=","
echo "----------开始读取城市列表文件----------"
cat $1 | while read line
do
    for city in $line
    do
            curl http://www.dianping.com/$city -I >> $RESULTFILE
    done
    echo "--------------结束一行的读取--------------"
done
echo "---------------结束所有的读取----------------"

深入分析Volatile

By EdwardGuo

發表於 2016-01-31

在进入volatile之前，咱们先普及几个术语
1.共享变量：在多个线程之间能够被共享的变量，包括实例变量，静态变量和数组元素，实例变量都被放在堆内存中，volatile只作用于共享变量。
2.内存屏障：memory barriers是一组指令，用于实现对内存操作的顺序限制。
3.缓冲行：Cache line，缓存中可以分配的最小存储单元，
4.原子操作：Atomic operations 不可中断的一个或一系列操作。
5.缓存行填充：cache line fill 当处理器识别到从内存中读取操作数是可以缓存的，处理器读取整个缓冲行到适当的缓存。
6.缓存命中：cache hit 如果进行高速缓存行填充的内存位置仍然是下次访问的地址时，处理器从缓存中读取操作数，而不是从内存中。
7.写命中：write hit 当处理器将操作数写回到一个内存缓存区域时，他会先检查这个缓存的内存地址是否在缓存行中，
如果存在一个有效的缓存行，则处理器会将这个操作数写入缓存而不是内存中

接下来看看在给一个共享变量加了volatile修饰之后会发生什么操作：
Java 代码： private volatile Boolean isShutdown;
汇编代码： 0x01a3de1d: movb $0×0,0×1104800(%esi);
0x01a3de24: lock addl $0×0,(%esp);
加了volatile休息是的共享变量，会多一行汇编代码–lock前缀，这个lock前缀在多核处理器下会发生2件事情：
1.将当前处理器缓存行的数据会写到系统内存。
2.这个会写内存的操作会引起其他CPU里缓存了该内存地址的数据失效。
由于在多核处理器中，为了提高处理器速度，处理器不直接和内存进行通讯，而是先将内存的数据读到内部缓存后再进行操作，操作后又写会缓存，但是不知道何时会写回内存，
这样其他处理器便看不到这个操作的改变，但是如果给一个变量加了volatile修饰，那么在进行写操作时，JVM虚拟机会给处理器发一条Lock前缀的指令，促发当前的写操作会
写至内存，会写之内存后，由于多核处理器实现了缓存一致性协议，每个处理器通过嗅探监测在总线上传播过来的数据来检查自己的缓存是否已失效，当处理器发现自己缓存行
的对应内存地址发生改变了，便会把处理器当前的缓存行置为已失效状态，同时将内存中的数据写入缓存。
术语太复杂，总结下来即为如下
给变量添加volatile修饰—>写操作时JVM发送Lock前缀指令—>当前的写操作回写至系统内存—>其他处理监听到总线的数据变化导致缓存行失效—>其他线程将系统
内存数据写入缓存—>从而保证数据一致性。

Do you really need to Buy a car

By EdwardGuo

發表於 2016-01-31

买车这个想法已经有了1年多了，拿到驾照也已经一年半了，从一开始想买新车到最近犹豫买二手车，缺一直没有衡量清楚买车的利与弊，我想大部分28，9岁的单身年轻人应该和我一样，在这个问题上纠结过，下面分析下利弊。
买车能带来什么好处？
1.最直接的好处满足开车的操作感，开车对年轻人来说也是一种乐趣，享受那种速度带来的激情。
2.生活品质的提高，同时带来便利，下雨天出门有个车会方便很多，不用为打不到车烦恼。
3.对单身朋友来说，有车可以为找对象加分，妹子需要车的时候可以提供便利。
4.回老家方便，同时回去也会有些面子。

以上是买车带来的好处，好像也想不出来更多了，下面看看买车带来的消费。

1.最直接的消耗是买车钱10万块左右，这10万块可能在买房的时候能起到关键作用。
2.其次一年油钱停车费加起来2万块，虽然不多，算算好像也不多。
3.停车难停车难停车难....虽然开车方便但是在上海停车实在太难.
4.堵车是另外一方面，交通太堵，即使买车了也只能周末开开，上下班可能都不能开车。

以上便是利弊，当然如果买辆保有量大的车，开个1，2年卖掉转手，成本可能就4，5万也不算太差，所以我觉得不用纠结太多，如果有停车的地方，那就买起来。

ClassNotFound VS NoClassDef

By EdwardGuo

發表於 2016-01-07

The difference between ClassNotFoundException and NoClassDefFoundError

--ClassNotFoundException
Thrown when an application tries to load in a class through its string name using 
    1) Class.forname() method in class Class.
    2) ClassLoader.findSystemClass method in class ClassLoader.
    3) ClassLoader.loadClass() in class ClassLoader.
As for ClassNotFoundException, it appears that it may stem from trying to make reflective calls to classes at runtime, but the classes the program is trying to call is does not exist.


--NoClassDefFoundError
Thrown if the JVM or a ClassLoader instance tries to load in the definition of a class but no definition could be found.
So, it appears that the NoClassDefFoundError occurs when the source was successfully compiled, but at runtime, the required class files were not found. This may be something that can happen in the distribution or production of JAR files, where not all the required class files were included.

2016年已至，再过2年就三十而立了，希望2年后自己再回首时，少些遗憾少些后悔，想清楚30岁之前最想实现的小幸运，
来说说最想做的10件事吧。

1.找到自己soul-mate，如果说刚毕业时没钱，毕业后又忙于工作，如今，工作已经4年半，我想2016年是时候遇见自己的灵魂伴侣了，
多一份执着，多一份包容，多一份稳重，少一些焦躁，少一些欲望，少一些骄傲。
2.将点评券系统打造成一个简洁健壮的系统，多花些时间补充知识，同时学会如何管理团队提高团队作战力，2016年成长为真正的
资深互联网工程师。
3.2016年下半年开始拍沪牌，如果拍中，买一辆高尔夫或者速腾。
4.30岁之前去一趟欧洲二战战场德国或波兰。
5.至少写6篇技术博客，作为一名技术工程师，学习新知识不能停止，博客是最好的总结归纳的手段。
6.带领团队实现一个技术或产品突破，争取拿到公司的最强音。
7.改掉晚睡的习惯，做到每天11点20前上床，11点半前睡觉。
8.减少刷朋友圈的次数，睡前看些技术或者床头灯读物或者听BBC而不是刷朋友圈。
9.开发一个简单IOS App。
10.在旅游淡季，开车带爹妈上海一周游。

以上更像是接下来两年要实现的愿望，加油吧少年，哦对了，还有最重要的一点，攒够首付！。
