Hash算法和HashMap

發表於 2016-12-18

  • 什么是哈希算法

    • 所谓的哈希算法即将某个东西(通常情况下是某个数或字符串)，经过某种特殊处理，得到某种形式的结果，说白了是一种高效的存储结构。

  • Hash有哪些构造方法

    • 直接寻址法：直接使用输入的关键字或者关键字的线性函数值作为哈希值，过于简单。

      • H(key) = key 或者H(key)=a*key+b

    • 除留余数法：将关键字对某个不大于地址空的数p取模，整除后得到的数即为散列地址，常用。

      • H(key) = key MOD p（p<=m）

  • 为啥要使用哈希算法呢？

    • 哈希能快速定位到记录要插入的位置，所以比较适用于要频繁插入的场景。

    • 哈希能快速定位到要查找记录的位置，所以比较适用于要频繁查找的场景。

    • 综上2点，比较适用于查找和插入频繁的场景，在所有互联网应用程序，插入和查找是必不可少的，所以哈希会如此受欢迎。

  • 哈希算法也有缺点，如下：

    • 元素的有序性被打破。

    • 存储空间消耗较大。

  • 来一个哈希实例—HashMap

    • HashMap 应该是java程序员最熟悉不过的数据结构，HashMap怎么利用哈希工作呢？

      • 一张图看下HashMap的内部结构

      • 从图可以看出HashMap是由数组+链表组成，容器就是个线性数组，每个数组的元素存的是链表，注意每个数组只存链表的头元素，并没有存整个链表。

      • HashMap的 put实现：

        • HashMap的所有内容存在一张Entry[] table 中,每次put会先找到key对应在数组中的index。

        • 找到了待插入元素的位置后，将当前该位置的Entry对象设置带插入对象的Next值，再将待插入对象放置该位置的Entry table中，也就是table数组中存储的是最后插入的数据

        • get()相对简单，不做分析。

      • HashMap扩容机制：当一个index的链表过长，以防影响性能，table会进行扩容，哪几种情况会扩容呢？

        • Entrytable 元素个数大小大于 table size个数*loadFactor，会进行扩容。

      • 如何解决哈希冲突？

        • 构造良好的哈希函数，比如被取模的数尽可能使用较大的素数。

        • 再散列法，当关键字key的哈希地址p=H(key) 出现冲突时，以p为基础，产生另一个哈希地址p1,如果p1依旧冲突继续哈希，直到找到不冲突的地址。

        • 链地址法，这就是HashMap使用的方法：这种方法的基本思想是将所有哈希地址为i的元素构成一个称为同义词链的单链表,并将单链表的头指针存在哈希表的第i个单元中，因而查找、插入和删除主要在同义词链中进行。链地址法适用于经常进行插入和删除的情况。
